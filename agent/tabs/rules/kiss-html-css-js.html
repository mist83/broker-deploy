KISS HTML/CSS/JS - Keep It Simple, Semantic

TRIGGER

When the user says "KISS the html", "KISS the css", "KISS the js", or variations like "keep it simple html", activate this rule to enforce minimalist, semantic web design principles.

PHILOSOPHY

This rule enforces a strict anti-bloat, anti-AI-stink approach to web development:

  • NO CSS gradients - Flat colors only
  • NO unnecessary animations - Only when functionally required
  • NO over-designed elements - Simple, clean, functional
  • NO "AI stink" - Avoid patterns that scream "generated by AI"
  • YES to semantic HTML - Use proper HTML5 elements
  • YES to minimalism - Less is more

CORE PRINCIPLES

HTML Guidelines

  1. Use semantic HTML5 elements:

    • <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>
    • <button> for buttons, not styled <div> elements
    • <a> for links, not JavaScript onclick handlers
    • <form> and proper input types

  2. Keep structure minimal:

    • Avoid unnecessary wrapper divs
    • No div soup - every element should have a purpose
    • Use the simplest structure that works

  3. Accessibility first:

    • Proper heading hierarchy (h1, h2, h3)
    • Alt text for images
    • Labels for form inputs
    • ARIA attributes only when semantic HTML isn't enough

CSS Guidelines

  1. Flat design only:

    • Solid colors, no gradients
    • Simple borders, no box-shadows unless absolutely necessary
    • No complex transforms or 3D effects

  2. Minimal styling:

    • System fonts or common web-safe fonts (Arial, sans-serif)
    • Simple color palette (2-3 colors max)
    • Consistent spacing using a simple scale (8px, 16px, 24px, 32px)
    • No excessive padding/margin

  3. No bloat:

    • No CSS frameworks unless specifically requested
    • No utility class explosion
    • Keep selectors simple and semantic
    • Avoid overly specific selectors

  4. Class consolidation:

    • Multiple classes serving identical purpose = ONE shared class
    • Keep unique IDs for JS targeting
    • Example: .chat-sidebar, .task-sidebar, .workspace-sidebar → .sidebar
    • HTML pattern: <div class="sidebar" id="chat-sidebar">

  5. No inline styles:

    • Extract repeated inline styles to CSS classes
    • Create utility classes for common patterns
    • Inline styles only for truly dynamic values

  6. Layout:

    • Use Flexbox or Grid for layouts, not floats
    • Mobile-first responsive design
    • Simple breakpoints (1-2 max)

JavaScript Guidelines

  1. Vanilla JS preferred:

    • No frameworks unless specifically requested
    • Use modern ES6+ features
    • Keep it functional and readable

  2. Minimal DOM manipulation:

    • Use event delegation
    • Avoid excessive DOM queries
    • Keep JavaScript separate from HTML (no inline handlers)

  3. Progressive enhancement:

    • Core functionality works without JS
    • JS enhances, doesn't enable

ANTI-PATTERNS TO AVOID

CSS Anti-Patterns (AI Stink)

❌ background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); ✅ background: #667eea;

❌ box-shadow: 0 10px 40px rgba(0,0,0,0.2); ✅ border: 1px solid #ddd;

❌ border-radius: 12px; (on everything) ✅ border-radius: 4px; (sparingly)

❌ Complex animations on hover ✅ Simple opacity or color change

HTML Anti-Patterns

❌ <div class="button-wrapper"><div class="button-inner"><button>Click</button></div></div> ✅ <button>Click</button>

❌ <div onclick="doSomething()">Click me</div> ✅ <button onclick="doSomething()">Click me</button> or better: <button id="myBtn">Click me</button> with event listener

❌ <div class="heading">Title</div> ✅ <h1>Title</h1>

JavaScript Anti-Patterns

❌ jQuery for everything ✅ Vanilla JS with clear, purposeful code

❌ Massive nested callbacks ✅ Async/await or promises

❌ Global variables everywhere ✅ Module pattern or ES6 modules

EXAMPLE: GOOD VS BAD

Bad (AI Stink)

<div class="hero-section">
  <div class="container">
    <div class="row">
      <div class="col-12">
        <div class="heading-wrapper">
          <h1 class="main-heading gradient-text animated fadeIn">Welcome</h1>
        </div>
        <div class="button-group">
          <div class="button-wrapper">
            <a href="#" class="btn btn-primary btn-lg rounded-pill shadow-lg">
              Get Started
            </a>
          </div>
        </div>
      </div>
    </div>
  </div>
</div>

Good (KISS)

<header>
  <h1>Welcome</h1>
  <a href="#" class="cta">Get Started</a>
</header>

header {
  padding: 2rem;
  text-align: center;
}

h1 {
  margin: 0 0 1rem 0;
  font-size: 2rem;
}

.cta {
  display: inline-block;
  padding: 0.75rem 1.5rem;
  background: #0066cc;
  color: white;
  text-decoration: none;
  border-radius: 4px;
}

.cta:hover {
  background: #0052a3;
}

IMPLEMENTATION CHECKLIST

When creating or modifying HTML/CSS/JS, verify:

HTML

  • Uses semantic HTML5 elements
  • Minimal nesting (3-4 levels max)
  • No unnecessary wrapper divs
  • Proper heading hierarchy
  • Accessible (labels, alt text, etc.)

CSS

  • Flat colors only, no gradients
  • Simple borders, minimal shadows
  • System fonts or web-safe fonts
  • Consistent spacing scale
  • No CSS framework bloat
  • Mobile-first responsive

JavaScript

  • Vanilla JS unless framework requested
  • Clean, readable code
  • Proper event handling
  • No inline handlers
  • Progressive enhancement

ENFORCEMENT

When the user invokes this rule:

  1. Audit existing code for violations
  2. Simplify - remove unnecessary elements, styles, scripts
  3. Flatten - remove gradients, complex shadows, animations
  4. Semanticize - replace divs with proper HTML5 elements
  5. Clean - remove bloated class names, unused styles
  6. Report what was simplified and why

TONE FOR THIS RULE

When enforcing KISS, be direct and pragmatic:

  • "Removed gradient - using flat color instead"
  • "Simplified markup - using semantic header element"
  • "Reduced nesting from 6 levels to 3"
  • "Removed unnecessary wrapper divs"

Don't apologize or ask permission - just execute the simplification and explain the improvements.

COLOR PALETTE SUGGESTIONS

When creating new designs, use simple, accessible color palettes:

Minimal Professional:

  • Background: #ffffff
  • Text: #222222
  • Primary: #0066cc
  • Border: #dddddd

High Contrast:

  • Background: #000000
  • Text: #ffffff
  • Accent: #00ff00
  • Secondary: #666666

Neutral Clean:

  • Background: #f5f5f5
  • Text: #333333
  • Primary: #2c3e50
  • Border: #cccccc

REMEMBER

The goal is functional simplicity. Every element, every style, every line of code should serve a clear purpose. If you can't explain why something is there in one sentence, it probably shouldn't be there.

Less code = Less bugs = More maintainable = Better user experience
